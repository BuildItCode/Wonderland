T2.1 — Phase Regression & Round Cap

2026-06-03 · task-complete · M2

Phases with Regression

stateDiagram-v2 [*] --> frame frame --> propose: advance propose --> implement: advance + consensus implement --> ratify: advance implement --> propose: fatal failure (round++) ratify --> propose: regress (round++) implement --> closed: round > cap = unsolvable ratify --> closed: declare(ratified) closed --> [*]

Rules

  • A fatal failure past the propose phase auto-regresses to propose.
  • Regression mints a fresh contract version (copy of the latest, signatures cleared, old marked superseded) — re-signature is forced before the room can advance again.
  • The facilitator can also regress explicitly; the reason is recorded in the transcript.
  • Each regression bumps the round; exceeding the template round cap (8) closes the room as unsolvable.
